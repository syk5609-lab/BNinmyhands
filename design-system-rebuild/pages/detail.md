# Detail Rebuild Rules

## 페이지 목표
디테일은 한 심볼이 같은 run에서 왜 surfaced 되었는지 설명하는 same-run analysis story다.

## 핵심 구조
1. compact header
2. back action + symbol hero
3. same-run context strip
4. score rail
5. latest + delta
6. why this coin
7. funding context
8. recent history
9. discussion
10. bottom sponsor
11. disclosure footer

## 강제 규칙
- detail은 위젯 더미가 아니라 분석 문서처럼 읽혀야 한다.
- funding은 장식 메타가 아니라 해석 블록이어야 한다.
- discussion은 분석 아래의 보조 레이어여야 한다.
- sponsor는 항상 맨 아래에 두고 Sponsored 라벨을 표시한다.
