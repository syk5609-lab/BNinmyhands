# Dashboard Page Override Seed

## Page goal
Help users narrow today’s 5–10 most worth-watching symbols quickly, then move into a same-run detail view.

## Composition
The dashboard should feel like one operational workspace, not a mosaic of detached product cards.

Suggested order:
1. global shell / header
2. run context + preset context
3. top highlights
4. bucket summary
5. rankings table as the main surface
6. secondary sponsor insertion only where it does not interrupt scanning

## Priority
The rankings table is the main workspace.
Everything above it should improve interpretation or navigation into the table.

## First viewport rules
- must not collapse into accidental one-column behavior on desktop
- should reveal context + meaningful analysis, not just decorative chrome
- should make it obvious what the user can scan or act on next

## Visual treatment
- use dividers, spacing, and grouped sections before cards
- keep preset row compact and intentional
- keep highlights informative, not flashy
- keep bucket summary secondary but legible

## Copy style
Use labels like:
- Latest run
- Strategy preset
- Top highlights
- Bucket summary
- Full rankings
- Updated
- Unavailable

Avoid homepage-style slogans.
